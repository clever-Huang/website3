在具体进入页面之后

采用了一个视频，固定位置后作为背景。
具体的实现方法是利用css，设置一个类，将背景视频位置固定而产生的效果，具体代码如下：

html:

<div class="v">
            <video class="f" id="bgvid" playsinline="" autoplay muted loop="">
                <source src="/img/伊之助_Trim.mp4" type="video/mp4">
            </video>
        </div>

css:

.f {
    position: absolute;
    top: 50%;
    left: 50%;
    min-width: 100%;
    min-height: 100%;
    width: auto;
    height: auto;
    z-index: -9999;
    -webkit-transform: translateX(-50%) translateY(-50%);
    transform: translateX(-50%) translateY(-50%);
    -webkit-transition: 1s opacity;
    transition: 1s opacity;
    background-color: skyblue;
}

.v {
    position: fixed;
    width: 100%;
    height: 100%;
    overflow: hidden;
    z-index: -10000;
    background-color: skyblue;
}

.v:before {
    content: "";
    position: absolute;
    width: 100%;
    height: 100%;
    display: block;

2.将mp3,以及大文件mp4托管到github的方法

在上传图片过程中，是直接将上传到github的一个new issues上，可以生成一个图片的绝对链接，在html网页中实现引用

但由于其大小不能超过10M，视频无法直接插入生成网址。

但在跳转子页面时，引用的其他几个子页面html的相对路径，按照逻辑，应该将整个文件夹打包后，利用相对路径可以实现引用，但到视频时总是出现404，查阅了一番没有找到解决办法。

采用了另外一个办法

（1）人物传背景视频

对于背景视频，其长度只有一分钟左右，只有几十MB的大小，我随即将其导入PR，删除音频并降低了一些画质和长度，便能够成功的达到10M一下，并且能够成功地托管在github上，人物传的视频问题便解决了

（2）音乐界面视频

音乐界面的视频，我是选择的在固定位置播放。由于该MV视频长度有四分钟左右，是不可能压缩到10M以内引用的。但我在B站发现了分享中，可以直接有现成的代码引用

随即问题解决

（2）音乐界面mp3的引用(未解决)

五. 总结

本次作业是一次充满趣味的设计。与传统的课程不同，这是需要我们自己去将零件按照自己的想法去组装成一个完整的机器。当我们有一个有意思的设计想法时，首先需要看能否利用已有的模板或者语法知识来实现，再者再去查看是否有新的组件需要学习，探索。真正地做到了玩中学，学中玩。

但是本次设计还是有一些遗憾，没有实现一些功能。但是在遇到问题，解决问题的过程中，我积累到了很多网页设计的经验，相信通过这些经验，能为我以后更深入的学习打下坚实的基础。
